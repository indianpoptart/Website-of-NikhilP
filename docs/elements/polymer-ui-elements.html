Polymer UI elements

Basic UI components

Note: These elements and their documentation are very early. The “source” links below point to commented code where you can learn more about how to use the element.

Install everything:

bower install Polymer/polymer-ui-elements

<polymer-ui-accordion> source demo 

bower install Polymer/polymer-ui-accordion
<link rel="import"
      href="bower_components/polymer-ui-accordion/polymer-ui-accordion.html">

polymer-ui-accordion is a simple accordion style element. It should be used in conjunction with polymer-ui-collapsible. By default only one collapsible stays open at a time. To have multiple collapsibles open set "multi" property on the accordion.

Example:

<polymer-ui-accordion selected="0">
  <polymer-ui-collapsible>
    <div class="polymer-ui-collapsible-header">Header 1</div>
    <div>....</div>
  </polymer-ui-collapsible>
  <polymer-ui-collapsible>
    <div class="polymer-ui-collapsible-header">Header 2</div>
    <div>....</div>
  </polymer-ui-collapsible>
  <polymer-ui-collapsible>
    <div class="polymer-ui-collapsible-header">Header 3</div>
    <div>....</div>
  </polymer-ui-collapsible>
</polymer-ui-accordion>

activateEvent

String

Inherited from polymer-selector:

The event that would be fired from the item element to indicate it is being selected.

Default: 'tap'

itemSelector

String

Inherited from polymer-selector:

This can be used to query nodes from the target node to be used for selection items. Note this only works if the 'target' property is set.

Example:

<polymer-selector target="{{$.myForm}}" itemsSelector="input[type=radio]"></polymer-selector>
<form id="myForm">
  <label><input type="radio" name="color" value="red"> Red</label> <br>
  <label><input type="radio" name="color" value="green"> Green</label> <br>
  <label><input type="radio" name="color" value="blue"> Blue</label> <br>
  <p>color = {{color}}</p>
</form>

Default: ''

multi

Boolean

Inherited from polymer-selector:

If true, multiple selections are allowed.

Default: false

selected

Object

Inherited from polymer-selector:

Gets or sets the selected element. Default to use the index of the item element.

If you want a specific attribute value of the element to be used instead of index, set "valueattr" to that attribute name.

Example:

<polymer-selector valueattr="label" selected="foo">
  <div label="foo"></div>
  <div label="bar"></div>
  <div label="zot"></div>
</polymer-selector>

In multi-selection this should be an array of values.

Example:

<polymer-selector id="selector" valueattr="label" multi>
  <div label="foo"></div>
  <div label="bar"></div>
  <div label="zot"></div>
</polymer-selector>

this.$.selector.selected = ['foo', 'zot'];

Default: null

selectedClass

String

Inherited from polymer-selector:

Specifies the CSS class to be used to add to the selected element.

Default: 'polymer-selected'

selectedIndex

Number

Inherited from polymer-selector:

In single selection, this returns the selected index.

Default: -1

selectedItem

Object

Inherited from polymer-selector:

Returns the currently selected element. In multi-selection this returns an array of selected elements.

Default: null

selectedModel

Object

Inherited from polymer-selector:

In single selection, this returns the model associated with the selected element.

Default: null

selectedProperty

String

Inherited from polymer-selector:

Specifies the property to be used to set on the selected element to indicate its active state.

Default: 'active'

target

Object

Inherited from polymer-selector:

The target element that contains items. If this is not set polymer-selector is the container.

Default: null

valueattr

String

Inherited from polymer-selector:

Specifies the attribute to be used for "selected" attribute.

Default: 'name'

polymer-activate

Inherited from polymer-selector:

Fired when an item element is tapped.

Event Payload:

  • detail Object
    • item Object

      the item element

polymer-select

Inherited from polymer-selector:

Fired when an item's selection state is changed. This event is fired both when an item is selected or deselected. The isSelected detail property contains the selection state.

Event Payload:

  • detail Object
    • isSelected Boolean

      true for selection and false for deselection

    • item Object

      the item element

transition

String

This property specifies the animation to play during a page transition.

If the property is a polymer-animation-group, it should contain at least two animations. The first animation will target the the page it's transitioning away from, and the second animation will target the page it's transitioning to.

If the property is a string, it should be the tag name of a custom element that extends from polymer-animation-group that fits the above requirements.

polymer-ui-icon is a 24x24 glyph expressed as a background-image.

Example:

<polymer-ui-icon src="star.png"></polymer-ui-icon>

Optionally can use other size like 32x32 by setting the attribute "size" to "32":

<polymer-ui-icon src="big_star.png" size="32"></polymer-ui-icon>

Polymer includes an icon set. The property "icon" can be used to specify which icon to use.

Example:

<polymer-ui-icon icon="menu"></polymer-ui-icon>

See polymer-ui-iconset on how to use your own icon set.

icon

String

Specifies the icon from the icon set.

Default: ''

size

String

Specifies the size of the icon.

Default: 24

src

String

The URL of an image for the icon.

Default: ''

polymer-ui-iconset allows users to define their own icon sets.

Example:

<polymer-ui-iconset id="my-icons" src="my-icons.png" width="96" iconsize="24"
    icons="location place starta stopb bus car train walk">
</polymer-ui-iconset>

The above will automatically register the icon set "my-icons" to the iconset database. To use the user-defined icon set, prefix the icon with the icon set e.g. "my-icons:clock"

Example:

<polymer-ui-icon-button icon="my-icons:car"></polymer-ui-icon-button>

<polymer-ui-menu> source demo 

bower install Polymer/polymer-ui-menu
<link rel="import"
      href="bower_components/polymer-ui-menu/polymer-ui-menu.html">

polymer-ui-menu is a polymer-selector with theme propagation. It styles to look like a menu and should be used in conjunction with polymer-ui-menu-item.

Example:

<polymer-ui-menu selected="0">
  <polymer-ui-menu-item icon="settings" label="Settings"></polymer-ui-menu-item>
  <polymer-ui-menu-item icon="dialog" label="Dialog"></polymer-ui-menu-item>
  <polymer-ui-menu-item icon="search" label="Search"></polymer-ui-menu-item>
</polymer-ui-menu>

The "selectedItem" property returns the currently selected item.

Example:

<polymer-ui-menu selected="0" selectedItem="{{item}}">
  <polymer-ui-menu-item icon="settings" label="Settings"></polymer-ui-menu-item>
  <polymer-ui-menu-item icon="dialog" label="Dialog"></polymer-ui-menu-item>
  <polymer-ui-menu-item icon="search" label="Search"></polymer-ui-menu-item>
</polymer-ui-menu>

<div>selected label: {{item.label}}</div>

The event "polymer-select" can also be used to listen for selection change.

Example:

<polymer-ui-menu selected="0" on-polymer-select="{{selectAction}}">
  <polymer-ui-menu-item icon="settings" label="Settings"></polymer-ui-menu-item>
  <polymer-ui-menu-item icon="dialog" label="Dialog"></polymer-ui-menu-item>
  <polymer-ui-menu-item icon="search" label="Search"></polymer-ui-menu-item>
</polymer-ui-menu>

...

selectAction: function(e, detail) {
  if (detail.isSelected) {
    var selectedItem = detail.item;
    ...
  }
}

<polymer-ui-menu-button> source demo 

bower install Polymer/polymer-ui-menu-button
<link rel="import"
      href="bower_components/polymer-ui-menu-button/polymer-ui-menu-button.html">

polymer-ui-menu-button is a polymer-ui-icon-button with a drop down menu that allows the user to select an option. The drop down menu is styled with an arrow pointing to the button, and can be positioned to the top or the bottom of the button with the valign property. The valign property aligns the menu to the left or right edge of the button.

Example:

<polymer-ui-menu-button selected="0">
  <polymer-ui-menu-item icon="settings" label="Settings"></polymer-ui-menu-item>
  <polymer-ui-menu-item icon="dialog" label="Dialog"></polymer-ui-menu-item>
  <polymer-ui-menu-item icon="search" label="Search"></polymer-ui-menu-item>
</polymer-ui-menu-button>

halign

String

Horizontally align the overlay with the button. Accepted values are ["left", "center", "right"].

icon

String

The icon to display.

opened

Boolean

Set to true to open the menu.

selected

Number

The index of the selected menu item.

valign

String

Display the overlay on top or below the button. Accepted values are ["top", "bottom"].

selection

Node

The selected menu item.

toggle ()

Toggle the opened state of the dropdown.

<polymer-ui-menu-item icon="favorite" label="Favorite" href="http://www.polymer-project.org/"></polymer-ui-menu-item>

Fired when an item's selection state is changed. This event is fired both when an item is selected or deselected. The isSelected detail property contains the selection state.

Event Payload:

  • detail Object
    • isSelected Boolean

      true for selection and false for deselection

    • item Object

      the item element

activateEvent

Inherited from polymer-selector:

The event that would be fired from the item element to indicate it is being selected.

Default: 'tap'

itemSelector

String

Inherited from polymer-selector:

This can be used to query nodes from the target node to be used for selection items. Note this only works if the 'target' property is set.

Example:

<polymer-selector target="{{$.myForm}}" itemsSelector="input[type=radio]"></polymer-selector>
<form id="myForm">
  <label><input type="radio" name="color" value="red"> Red</label> <br>
  <label><input type="radio" name="color" value="green"> Green</label> <br>
  <label><input type="radio" name="color" value="blue"> Blue</label> <br>
  <p>color = {{color}}</p>
</form>

Default: ''

multi

Boolean

Inherited from polymer-selector:

If true, multiple selections are allowed.

Default: false

selected

Object

Inherited from polymer-selector:

Gets or sets the selected element. Default to use the index of the item element.

If you want a specific attribute value of the element to be used instead of index, set "valueattr" to that attribute name.

Example:

<polymer-selector valueattr="label" selected="foo">
  <div label="foo"></div>
  <div label="bar"></div>
  <div label="zot"></div>
</polymer-selector>

In multi-selection this should be an array of values.

Example:

<polymer-selector id="selector" valueattr="label" multi>
  <div label="foo"></div>
  <div label="bar"></div>
  <div label="zot"></div>
</polymer-selector>

this.$.selector.selected = ['foo', 'zot'];

Default: null

selectedClass

String

Inherited from polymer-selector:

Specifies the CSS class to be used to add to the selected element.

Default: 'polymer-selected'

selectedIndex

Number

Inherited from polymer-selector:

In single selection, this returns the selected index.

Default: -1

selectedItem

Object

Inherited from polymer-selector:

Returns the currently selected element. In multi-selection this returns an array of selected elements.

Default: null

selectedModel

Object

Inherited from polymer-selector:

In single selection, this returns the model associated with the selected element.

Default: null

selectedProperty

String

Inherited from polymer-selector:

Specifies the property to be used to set on the selected element to indicate its active state.

Default: 'active'

target

Object

Inherited from polymer-selector:

The target element that contains items. If this is not set polymer-selector is the container.

Default: null

valueattr

String

Inherited from polymer-selector:

Specifies the attribute to be used for "selected" attribute.

Default: 'name'

<polymer-ui-splitter> source demo 

bower install Polymer/polymer-ui-splitter
<link rel="import"
      href="bower_components/polymer-ui-splitter/polymer-ui-splitter.html">

polymer-ui-splitter provides a spilt bar and dragging on the split bar will resize the sibling element. Use the property "direction" to indicate which sibling to be resized and the orientation.

Example:

<div class="flexbox">
  <div>left</div>
  <polymer-ui-splitter direction="left"></polymer-ui-splitter>
  <div flex>right</div>
</div>

For horizontal splitter set direction to "up" or "down".

Example:

<div class="flexbox column">
  <div>top</div>
  <polymer-ui-splitter direction="up"></polymer-ui-splitter>
  <div flex>bottom</div>
</div>

direction

String

Possible values are "left", "right", "up" and "down".

Default: 'left'

locked

Boolean

Locks the split bar so it can't be dragged.

Default: false

    <polymer-ui-menu-item label="Topics 2"></polymer-ui-menu-item>
  </polymer-ui-submenu-item>
  <polymer-ui-submenu-item icon="settings" label="Favorites">
    <polymer-ui-menu-item label="Favorites 1"></polymer-ui-menu-item>
    <polymer-ui-menu-item label="Favorites 2"></polymer-ui-menu-item>
    <polymer-ui-menu-item label="Favorites 3"></polymer-ui-menu-item>
  </polymer-ui-submenu-item>
</polymer-ui-menu>

href

String

Inherited from polymer-ui-menu-item:

Specifies the URL of the link it goes to when tapped on.

Example:

<polymer-ui-menu-item icon="favorite" label="Favorite" href="http://www.polymer-project.org/"></polymer-ui-menu-item>

If you want more control on the link, e.g. specify the target for where to open the linked document, you can put <a> directly inside the menu-item.

Example:

<polymer-ui-menu-item icon="favorite" label="Favorite">
  <a href="http://www.polymer-project.org/" target="_self"></a>
</polymer-ui-menu-item>

Default: ''

<polymer-ui-toolbar> source demo 

bower install Polymer/polymer-ui-toolbar
<link rel="import"
      href="bower_components/polymer-ui-toolbar/polymer-ui-toolbar.html">

polymer-ui-toolbar is a horizontal bar containing elements that can perform actions.

Example:

<polymer-ui-toolbar>
  <polymer-ui-icon-button src="menu.png" on-click="{{menuAction}}"></polymer-ui-icon-button>
  <div flex>Title</div>
  <polymer-ui-icon-button src="more.png" on-click="{{moreAction}}"></polymer-ui-icon-button>
</polymer-ui-toolbar>

polymer-ui-toolbar can adopt to smaller screen size. If the attribute "responsive" is set and the screen size is less than the responsiveWidth (default to 800px), the toolbar will be moved to the bottom of the page.

Example:

<polymer-ui-toolbar>
  <polymer-ui-icon-button icon="menu"></polymer-ui-icon-button>
  <div flex>Title</div>
  <polymer-ui-toolbar responsive>
    <polymer-ui-icon-button icon="add"></polymer-ui-icon-button>
    <polymer-ui-icon-button icon="trash"></polymer-ui-icon-button>
    <polymer-ui-icon-button icon="search"></polymer-ui-icon-button>
  </polymer-ui-toolbar>
</polymer-ui-toolbar>

responsiveWidth

String

Max-width when element becomes responsive.

Default: '800px'
